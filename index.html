Что такое Open Evolution?

Open Evolution - это программа с открытым исходным кодом позволяющая моделировать развитие экосистемы животных на заданной решетке ячеек.

Модель подразумевает два вида ячеек - траву и животных. Трава имеет один параметр - степень роста. Животные имеют пять параметров - уровень, скорость, утомляемость, текущий запас энергии, максимальный запас энергии. При каждой итерации будет происходить 3 процесса:

  • Перемещение.
    Животные перемещаются на расстояние превосходящее по значению параметр скорости. После перемещения животные количество энергии равное параметру утомляемости. Если запас энергии животного не превышает ноль, то оно умирает.
  • Кормление.
    Животные восполняют затраченную энергию пищей. Животные с уровнем 0 - травоядные, остальные хищники. Травоядные животные едят траву и восполняют энергию пропорционально степени роста травы. Хищники поедают животных меньшего уровня и восстанавливают фиксированное значение энергии.
  • Размножение.
    Если в одной ячейки находится несколько особей одного вида, то они дают потомство. Параметры потомков определяются генетическими алгоритмами. При размножении родители теряют часть запаса энергии. Так же в эту стадию трава увеличивает свою степень роста. Если степень роста уже максимальна, то в соседних свободных ячейках появляется трава с минимальной степенью роста.

Основная идея - эволюция животных в условиях конкуренции как с особями своего вида, так и с особями других видов. В ходе такой конкуренции наименее приспособленные особи должны вымирать. Пользователь может изменять состояние ячеек во время моделирования. Так же пользователю будет предоставлена статистика развития популяций. Приложение выполнено в объекто-ориентированном стиле с клиент-серверной архитектурой. Сервер отвечает за логику приложения, клиент отвечает за предоставление информации пользователю, взаимодействии с ним и передачи полученных данных для обработки на сервер.

Как работает Open Evolution?

Приложение написано на языке C++ с использованием фреймворка Qt. Приложение выполнено в объектно-ориентированном стиле с применением паттерна проектирования "Посредник". Такой подход позволяет обеспечить модульную структуру программы и облегчить процесс новых модулей.

Клиенсткая часть приложения отвечает за обмен информацией с пользователем. Серверная часть отвечает за исполнение правил симуляции. Использование формата JSON для передачи данных позволяет избавиться от привязки к реализациям структур данных. По этому код серверного приложения можно переписать под использование других библиотек/фреймворков - изменения не должны коснутся клиентской части.

Будет ли Open Evolution развиваться дальше?

В будущем планируется расширить список параметров животных. Это позволит сделать симуляцию более реалистичной.

Так же для более осмысленного поведения животных при перемещении планируется использовать методы машинного обучения.

Для использования приложения с несеолькими серверами планируется реализоовать систему распределения нагрузки между ними.
